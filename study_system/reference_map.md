# 参考资源与学习方法 Reference Map

## 这页是干什么的

这页不是“书单堆砌”。
它的作用是帮你在不同阶段知道：

- 先学什么
- 用什么方式学更快
- 哪些资料适合扫盲
- 哪些资料适合补原理
- 哪些内容一定要配实验

## 最适合你的学习配比

建议你按这个比例来学：

- 20%：快速理解材料
- 40%：自己动手实验
- 20%：工具观察与 profiling
- 20%：复盘与整理

如果一上来就纯看书，速度会慢很多。
对你最有效的方式通常是：

`先有问题 -> 快速建模 -> 做实验 -> 用工具观察 -> 再回头补原理`

## 第一阶段：Linux 与系统基础

适合的学习材料：

- 入门解释类文章或视频
- man page
- 小型 demo 代码
- 观察工具官方说明

学习重点：

- 不求一开始全部学深
- 先建立关键词地图
- 每个主题都尽量配一个小实验

推荐实验方向：

- 单线程 vs 多线程
- 不同锁粒度对吞吐的影响
- `mmap` vs 普通读写
- cache locality 对数组遍历的影响
- `perf` / `strace` / `pidstat` 的基本使用

## 第二阶段：体系结构与性能

适合的学习材料：

- 面向工程师的体系结构资料
- 性能分析文章
- 结合 C/C++ 的示例代码

学习重点：

- 不要只记定义
- 要能把 cache、bandwidth、false sharing 和你的图像处理任务联系起来

推荐实验方向：

- 连续访问 vs 跨步访问
- AoS vs SoA
- false sharing 实验
- 线程数变化对吞吐和时延的影响

## 第三阶段：相机系统工程

适合的学习材料：

- 你们现有系统的真实链路
- 芯片与驱动文档
- 相机链路图
- 线上问题复盘

学习重点：

- 一定要围绕真实工作场景
- 每次都用“谁生产、谁拥有、谁等待、谁阻塞”去拆链路

推荐任务：

- 画一条真实 frame lifecycle
- 标出 copy 点和 queue 点
- 写一份延迟预算草图
- 给掉帧问题写排查模板

## 第四阶段：边缘部署

适合的学习材料：

- ONNX / TensorRT / 平台工具链官方资料
- 一个开源模型的真实部署过程
- runtime profiling 输出

学习重点：

- 目标不是训练模型
- 目标是看懂部署链路和系统瓶颈

推荐任务：

- 画一条模型部署路径
- 写一份 FP16 vs INT8 决策表
- 记录 preprocess / inference / postprocess 的时间拆分

## 第五阶段：机器人视觉迁移

适合的学习材料：

- ROS2 基础教程
- 相机到机器人坐标的系统图
- 标定和手眼标定的可视化说明

学习重点：

- 不要一开始就卷复杂算法
- 先学系统结构、消息流和标定思维

推荐任务：

- 画一个 ROS2 风格视觉链路
- 做一张标定症状-原因对照表
- 选一个小项目方向写 brief

## 怎么判断自己真的学会了

一个知识点如果满足下面 4 条，基本算真正进入长期记忆了：

1. 你能不用原文，自己讲清楚
2. 你能给出一个工作里的例子
3. 你能设计一个最小实验
4. 你能说出这个知识点最容易被误解的地方

## 最值得反复做的动作

- 画图
- 做最小实验
- 记录观察结果
- 用自己的话重写
- 一周后关掉资料重答
