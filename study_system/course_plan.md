# 课程总表 Course Plan

## 课程定位

这不是一套泛泛的计算机基础课。
它是专门围绕你的工作背景设计的个人学习路径：

- 相机应用开发
- 相机数据接入与链路调试
- 芯片接口与系统联调
- 想避开纯算法竞争，转向更稳的系统型能力

这套课的主线是：

`相机系统工程 -> Linux 系统能力 -> 性能与体系结构 -> 边缘 AI 部署 -> 机器人视觉迁移`

## 学完以后，希望你能做到什么

学完整条路线后，你应该逐步具备这些能力：

1. 能把相机与图像链路的主要瓶颈讲清楚。
2. 能用 Linux 工具定位常见性能问题，而不是只凭感觉。
3. 能从线程、内存、cache、拷贝、调度和 backpressure 角度解释系统行为。
4. 能理解边缘部署的真实工程问题，而不是只停留在“模型能不能跑”。
5. 能做出一个对外也拿得出手的小型系统项目。

## 这套课的设计原则

这套课程参考了现在比较成熟的学习网站常见结构，但内容完全围绕你个人定制。

每节课都尽量包含：

1. 为什么这节课对你的工作重要
2. 核心概念和心智模型
3. 一个实验、任务或观察练习
4. 一个可留下来的产物
5. 一组复盘问题

重点不是把知识“看完”，而是把知识变成：

- 直觉
- 调试习惯
- 复用能力
- 可展示的工程思考

## 推荐节奏

对于全职工程师，最实用的节奏是：

- 工作日：30 到 40 分钟
- 周末：2 到 4 小时

每节课建议拆成 4 次学习：

### 第 1 次

- 看课程
- 建立概念图
- 标出最模糊的点

### 第 2 次

- 做实验或任务
- 记录观察结果

### 第 3 次

- 用一个真实工作场景去连接这节课
- 写出你的系统解释

### 第 4 次

- 关掉资料，凭记忆回答复盘题
- 写自己的总结

## 三条能力线

### 能力线 A：Linux 与系统基础

目标：
学会解释运行时行为，建立排障视角。

### 能力线 B：相机系统与边缘部署

目标：
把你的相机背景和部署工程能力连接起来。

### 能力线 C：机器人视觉迁移

目标：
给你准备一条中长期的第二曲线。

## 阶段总览

## 第一阶段：系统基础

目标：
理解为什么系统会慢、会抖、会不稳定。

阶段产物：

- 一份 Linux 排障清单
- 一份关于线程/内存/cache 的理解笔记

## 第二阶段：相机与部署工程

目标：
把相机链路、运行时行为和部署瓶颈看成一个整体。

阶段产物：

- 一张相机到推理输出的 bottleneck map
- 一份部署决策矩阵

## 第三阶段：机器人视觉迁移

目标：
补齐 ROS2、标定和项目化能力。

阶段产物：

- 一张 ROS2 风格系统图
- 一份端到端项目说明书

## 每节课学完应该掌握什么

### 第 1 课：进程、线程、锁与上下文切换

重点：
共享状态、等待、争用、调度。

你应该能：

- 用自己的话解释进程和线程
- 区分 race、contention、deadlock、oversubscription
- 把上下文切换和时延抖动联系起来

### 第 2 课：虚拟内存、Page、Page Fault 与 `mmap`

重点：
内存布局、页行为、图像 buffer 的搬运成本。

你应该能：

- 解释堆、栈、映射内存的区别
- 理解 page fault 在系统中的位置
- 解释为什么大 buffer 拷贝很容易成为瓶颈

### 第 3 课：CPU Cache、局部性与内存访问

重点：
cache line、局部性、false sharing。

你应该能：

- 用直觉解释 cache 为什么重要
- 识别常见的 false sharing 风险
- 解释数据布局为什么会影响图像处理性能

### 第 4 课：Linux 观测与排障

重点：
`top`、`pidstat`、`vmstat`、`strace`、`perf`。

你应该能：

- 根据症状选择第一个工具
- 区分 CPU 忙、等待多、内存紧张这几类问题
- 建立自己的初版排障流程

### 第 5 课：相机数据通路、Buffer 与帧生命周期

重点：
frame ownership、queue、copy、backpressure。

你应该能：

- 画清楚一帧图像的真实流动路径
- 标出 buffer 的归属关系
- 找出最可能的隐藏拷贝点和排队点

### 第 6 课：延迟、抖动、掉帧与同步

重点：
latency、throughput、tail latency、jitter、sync policy。

你应该能：

- 区分平均延迟和 tail latency
- 解释为什么 deeper queue 不一定是好事
- 写出一个掉帧诊断流程

### 第 7 课：ONNX 与运行时基础

重点：
模型导出、运行时执行、preprocess / inference / postprocess。

你应该能：

- 描述一个模型从源框架到 runtime 的路径
- 找出非模型本体的开销
- 用系统工程视角看部署

### 第 8 课：量化、TensorRT 与工具链

重点：
FP16 / INT8、calibration data、toolchain choice。

你应该能：

- 解释量化的真实价值和风险
- 比较两个部署方案的工程差异
- 写出自己的部署决策矩阵

### 第 9 课：部署 Profiling

重点：
latency budget、瓶颈分类、tail 行为。

你应该能：

- 拆出一条部署路径的延迟预算
- 区分 compute-bound、memory-bound、transfer-bound
- 写出合理的 profiling 顺序

### 第 10 课：ROS2 基础

重点：
node、topic、QoS、launch。

你应该能：

- 画出 ROS2 风格的相机链路
- 解释 QoS 为什么对视觉数据重要
- 识别哪些模块适合拆节点，哪些适合留在进程内

### 第 11 课：标定、手眼标定与 3D 视觉

重点：
intrinsics、extrinsics、误差传播、3D setup。

你应该能：

- 解释内参和外参的工程意义
- 把标定误差和下游系统症状对应起来
- 理解为什么 3D 视觉更依赖安装和标定质量

### 第 12 课：端到端项目设计

重点：
项目范围、架构、观测性、可展示性。

你应该能：

- 给自己定一个范围合适的小项目
- 定义里程碑、指标和交付物
- 把项目写成公开作品

为了让课程更完整，后续我们还可以继续补这些内容：

- 每课术语表 Glossary
- 每课“工作现场案例”
- 每课“常见误区”
- 每课“面试/转岗会怎么问”
- 每课“延伸实验”
- 一条完整的项目制主线

## 最值得保留的作品

学完整条路线后，尽量留下这些产物：

1. 一份 Linux 排障清单
2. 一份内存或 cache 实验记录
3. 一张相机数据通路图
4. 一份 latency budget 表
5. 一份部署决策矩阵
6. 一张 ROS2 风格系统图
7. 一份项目 brief 或 demo 仓库

这些东西对职业成长的价值，通常比“我看过很多资料”更高。
