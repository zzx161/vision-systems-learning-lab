# Integrated Learning Route

## Positioning

Your goal is not to become a pure algorithm engineer.

Your target role is:

- camera/system engineer with strong Linux and performance fundamentals
- able to debug real hardware and data paths
- able to deploy AI models on edge platforms
- able to transfer into robotics/industrial vision if needed

This route combines four strengths:

1. Camera and imaging pipeline understanding
2. Linux systems and computer architecture
3. Edge AI deployment and performance engineering
4. Robotics vision system awareness

## Why This Route

Pure coding is increasingly automated.
Pure algorithm roles are increasingly crowded.
But the layer that connects hardware, runtime systems, data movement, deployment, debugging, and field reliability remains hard to replace.

That is the layer we will optimize for.

## The 4 Main Tracks

### Track A: Linux Systems and Architecture
This is your foundation.

What you will gain:

- stronger debugging ability
- better performance intuition
- deeper understanding of latency, memory, and throughput
- stronger confidence when facing camera or deployment bottlenecks

Modules:

1. Processes and threads
2. Virtual memory
3. IO and zero-copy intuition
4. Scheduling and latency
5. Linux observability tools
6. Cache and memory hierarchy
7. Multi-core and contention
8. SIMD and data-oriented thinking

### Track B: Camera and Data Path Engineering
This deepens your existing advantage.

What you will gain:

- stronger ownership of the full camera path
- ability to diagnose frame drop, latency jitter, bandwidth issues
- ability to think like a system owner, not only a feature implementer

Modules:

1. Sensor to ISP to memory path
2. Buffering and frame lifecycle
3. Timing, synchronization, and stability
4. Data movement cost and copy reduction
5. Fault isolation and production debugging

### Track C: Edge AI Deployment
This is not algorithm training.
This is model delivery and systems integration.

What you will gain:

- ability to bridge camera systems and deployed AI
- practical value on embedded and automotive platforms
- better career flexibility across AD, robotics, and industrial vision

Modules:

1. ONNX basics
2. Inference engine basics
3. Quantization intuition
4. TensorRT or platform-specific toolchain basics
5. Profiling and deployment bottlenecks
6. Accuracy-latency tradeoff thinking

### Track D: Robotics Vision Transition Ability
This is your second curve.

What you will gain:

- broader market mobility
- transferability into robotics and industrial automation
- system-level perspective beyond automotive imaging

Modules:

1. ROS2 basics
2. Camera calibration and hand-eye calibration awareness
3. 3D vision basics
4. SLAM framework awareness
5. Small practical project integration

## Recommended Order

### Stage 1: Build the Foundation
Duration: 3 months

Focus:

1. Linux systems
2. computer architecture
3. camera data path thinking

Goal:

Be able to explain and diagnose:

- thread contention
- memory copying cost
- CPU spikes
- latency jitter
- frame drop causes

### Stage 2: Add Edge Deployment
Duration: 3 to 6 months

Focus:

1. ONNX
2. inference runtime basics
3. quantization
4. profiling
5. deployment on embedded targets

Goal:

Take a simple model from export to deployment and understand why it is slow or unstable.

### Stage 3: Build a Second Curve in Robotics
Duration: 6 to 12 months

Focus:

1. ROS2
2. calibration awareness
3. 3D vision intuition
4. small end-to-end project

Goal:

Build transferable system ability for robotics and industrial vision.

## Weekly Time Budget

For a full-time engineer, a realistic rhythm is:

- 30 minutes on workdays
- 2 to 4 hours on weekends

## Learning Method

For every module:

1. concept overview
2. one small experiment
3. one debugging/tool practice
4. one written summary
5. one review after one week

## Role Target After This Route

Examples of target titles:

- camera system engineer
- perception infrastructure engineer
- edge AI deployment engineer
- vision runtime/performance engineer
- robotics vision system engineer
