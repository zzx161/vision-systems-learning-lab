# 相机链路排查模板

## 适用场景

适合排查这些问题：

- 掉帧
- 延迟变大
- 输出不稳定
- 多路相机同步差
- 某条链路吞吐上不去

## 第一步：先画一帧的旅程

先把一帧数据的主要路径写出来：

1. 从哪里来
2. 第一次落在哪块 buffer
3. 中间经过哪些阶段
4. 在哪几处 copy
5. 在哪几处排队
6. 在哪里释放

## 第二步：列出关键角色

对每个阶段写：

- 生产者是谁
- 消费者是谁
- buffer 归属是谁
- 等待点在哪里

## 第三步：优先查这 5 类问题

1. hidden copy
2. queue 过深
3. buffer 回收不及时
4. 下游 backpressure
5. 同步策略导致丢帧

## 第四步：必须补的时间戳

如果现在没有完整时间信息，优先补：

1. 采集时间
2. 进入处理时间
3. 处理完成时间
4. 进入下游时间
5. 输出时间

## 第五步：如果掉帧，先判断是哪一类

### 类型 A：算不过来

表现：

- 长期积压
- 平均值也差

### 类型 B：偶发抖动

表现：

- 平均值还行
- 但偶发 spike 很大

### 类型 C：策略丢弃

表现：

- 系统主动丢弃旧帧
- 同步对齐失败

## 一页速记

- 先画路径，再猜问题
- 先看 buffer 归属，再看 copy
- 先看 queue，再看下游 backpressure
- 先区分“算不过来”还是“偶发抖动”
