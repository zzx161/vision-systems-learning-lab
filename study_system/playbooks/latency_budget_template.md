# Latency Budget 模板

## 怎么用

把一条链路拆成若干段，不要只记录总时间。

| 阶段 | 平均时间 | 最坏时间 | 抖动情况 | 备注 |
| --- | --- | --- | --- | --- |
| 输入获取 |  |  |  |  |
| 预处理 |  |  |  |  |
| 数据传输 |  |  |  |  |
| 主要处理 / 推理 |  |  |  |  |
| 后处理 |  |  |  |  |
| 输出 |  |  |  |  |

## 填写时要注意

1. 初始化时间和稳态时间分开记
2. 平均值和最坏值都要记
3. 如果是相机系统，要额外记录“帧年龄”
4. 如果是部署链路，要单独看 preprocess 和传输

## 结论模板

你最后至少要能写出：

- 最大瓶颈在哪一段
- 最不稳定的是哪一段
- 哪一段最值得先优化
