# 课程环境依赖审查

## 触发原因

第 5 课讲到 J6 相机链路时，内容过早进入真实硬件、SDK、buffer handle、模块边界验证。
当前没有板子、日志和 SDK 环境，因此这些内容无法执行，也无法验证。

问题不是概念本身没价值，而是教学方式没有先判断环境条件。

## 以后带课的规则

1. 先判断这节课是否需要真实环境。
2. 没有环境时，只讲能被当前条件支持的内容。
3. 把真实环境内容写成以后可执行的检查清单。
4. 不把无法验证的链路判断当作学习成果。
5. 优先提供本地实验、模拟程序、配置样例或纸面模板。

## 已识别的高环境依赖课程

| 课程 | 风险 | 当前学习方式 |
| --- | --- | --- |
| 第 5 课 IO/DMA/zero-copy | 容易空讲硬件链路 | 学概念边界和证据清单，不判断真实 J6 copy |
| 第 7 课相机数据通路 | 需要 SDK/buffer 日志 | 先画帧生命周期模板 |
| 第 8 课延迟/掉帧/同步 | 需要 frame id 和时间戳 | 先用纸面时间线或模拟日志 |
| 第 10 课 ONNX/runtime | 需要模型和输入样例 | 先学输入输出契约 |
| 第 11 课量化/工具链 | 需要模型、校准集、目标平台 | 先学决策矩阵和风险 |
| 第 12 课 inference profiling | 需要可运行推理链路 | 先学 latency budget |
| 第 13 课 ROS2 | 需要 ROS2 环境才做命令 | 先画拓扑 |
| 第 14 课标定/3D | 需要数据集或设备 | 先学坐标系和误差来源 |

## 对第 5 课的修正

第 5 课当前不应按“完成现场排查能力”收尾。
更合适的当前完成标准：

1. 能解释 copy、mmap、DMA、zero-copy 的边界。
2. 能说明为什么没有环境时不能断言某段真实链路是否 copy。
3. 能写出上板后要收集的证据：buffer id、fd、format、stride、timestamp、BPU 输入准备耗时。

## 下一步

第 5 课应重讲为：

1. 本地可验证部分：复用第 2 课 `read/mmap/copy` 实验。
2. 纸面账本部分：只画抽象路径，不断言真实 J6 内部行为。
3. 证据清单部分：列出未来在 J6 环境中要怎么验证。
